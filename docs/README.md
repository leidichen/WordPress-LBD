# WordPress Dear 主题开发文档

本文档用于支持主题的持续开发与新人快速接手。目标是“少而精”：只保留真正指导开发的内容，不再维护碎片化的过程日志。

## 阅读顺序（新人建议）

1. [01-新人上手](./01-新人上手.md)
2. [02-目录与架构](./02-目录与架构.md)
3. [03-核心功能说明](./03-核心功能说明.md)
4. [04-开发规范](./04-开发规范.md)
5. [05-常见问题排查](./05-常见问题排查.md)

## 文档范围

- 面向本主题当前代码状态（`v1.2.1` 基础上的定制版本）。
- 优先记录“现状事实”和“可执行步骤”。
- 历史过程型细节已合并为排障与实现说明，不再单独存档。

## 维护规则

- 修改主题功能后，优先更新对应章节，不新增“某次修复日志”类文档。
- 一项信息只保留一处权威来源，避免重复描述。
- 新增模板、设置项、钩子时，必须同步更新 `02` 和 `03` 文档。
