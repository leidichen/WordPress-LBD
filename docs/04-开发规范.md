# 04｜开发规范

## 代码规范

- PHP：遵循 WordPress 常见写法，输出内容优先使用转义函数。
- CSS：统一使用变量驱动主题色，避免散落硬编码。
- JavaScript：仅保留必要交互，避免全局污染。

## 文件改动规范

- 改动功能前，先确认入口文件与影响页面。
- 修改模板时，同步检查移动端断点表现。
- 不在主题目录保留临时文件（如 `*_new.php`、`*_bak.php`）；如确有需要，合并后及时清理。

## 文档规范

- 文档标题与正文统一中文。
- 只维护“现状文档”，不新增“过程日志型”文件。
- 功能变更必须更新 `03-核心功能说明.md`。

## 测试规范

每次改动至少覆盖：

- 主题切换：默认态、切换态、刷新持久化
- 页面类型：首页、单篇、分类、搜索、404
- 闪念：无图/多图、短文本/长文本、展开/收起
- 周刊：不同列数、不同间距、移动端展示

## 提交规范（建议）

- 一次提交只做一类改动（功能或重构或样式）。
- 提交说明格式建议：`模块: 动作 + 结果`
- 示例：`闪念: 修复多图轮播首图索引错误`
