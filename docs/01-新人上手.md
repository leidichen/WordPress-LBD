# 01｜新人上手

## 目标

在最短时间内完成：环境准备、主题关键行为理解、常见改动落地。

## 第一步：确认运行环境

- WordPress 已可正常访问。
- 已激活主题：`Dear`。
- 主题目录：`wp-content/themes/wordpress-dear-1.2.0`。

## 第二步：理解页面入口

- 首页与单篇文章入口：`index.php`
- 站点头部（导航、主题切换按钮）：`header.php`
- 归档/分类页：`archive.php`
- 搜索模板：`search.php`
- 博客独立页面模板：`blog.php`
- 闪念独立页面模板：`idea.php`
- 周刊独立页面模板：`weekly.php`

## 第三步：先读这 3 个核心文件

1. `functions.php`
作用：主题注册、脚本样式加载、自定义设置、查询改写、性能优化。

2. `assets/css/base.css`
作用：全站样式变量与基础样式，亮暗主题切换依赖此文件中的 CSS 变量。

3. `idea.php`
作用：闪念卡片页面的结构与样式，含展开、图片查看与交互相关呈现。

## 第四步：本主题最常见改动入口

- 修改首页“近期文章”上方内容：`index.php` 中 `$article_id`
- 修改默认字体方案：`assets/css/base.css` 中 `body { font-family: ... }`
- 调整闪念显示参数：后台“自定义”中的“闪念设置”
- 调整周刊网格与间距：后台“自定义”中的“周刊设置”

## 第五步：本地改动自检清单

- 首页、文章页、归档页、搜索页是否均可打开。
- 亮暗主题切换是否正常、刷新后状态是否保持。
- 闪念页在有图/无图、短文/长文下显示是否正常。
- 移动端（宽度 < 768px）布局是否正常。

## 交接要求

提交代码前，至少同步更新：

- `03-核心功能说明.md`（功能行为变更）
- `05-常见问题排查.md`（新增高频问题）
